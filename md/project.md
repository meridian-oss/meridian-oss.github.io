# Meridian計画について

## 1. Purpose / プロジェクトの目的

Meridian計画は, ロボット開発を入り口に, 基礎ゼロから最先端技術へとつながる**継続可能な学習スロープ**を社会に実装することを目的とします.  

現在, 科学や工学には豊富な技術資産が存在しますが, それらを横断的に活用できる共通基盤が不足しています.  結果として, 初期コストや学習コストが高く, 参入障壁となっています. 

Meridianは, ロボットの状態データをリアルタイムにデバイス間・ソフトウェア間で共有する中間プロトコルを提供します. これにより, 既存のハードウェアやソフトウェア資産をシームレスに連携させ, 低コストかつ低学習コストでロボット開発を始められる環境を実現します. 

目指すのは, 科学を一部の専門家のものにとどめず, 生涯にわたって楽しめる知的娯楽へと再定義することです. 
その結果として, 科学に触れる人口を拡大し, 社会全体の技術力の底上げに貢献します.  

**Meridianは, 誰もが最先端技術に挑戦できる土壌を広げるプロジェクト**です.

## 2. Scope / プロジェクトの範囲

Meridian計画で規定するのは, 次の範囲です. 

- ロボット**状態データの表現**やルール
- デバイス間・ソフトウェア間でのデータの**リアルタイム共有の方式**
- 互換性を担保するための**最小要件**

これらは, 特定のロボット機体や特定のベンダーに依存しないことを目指します.

## 3. System / システム
**Meridian**は当開発における仕様およびシステム全体を指します.
Meridianの中間プロトコル仕様に則ったデータ配列を**Meridim**とし, その中間プロトコルを基軸としたエコシステムを**Meridian Flow System**とします. 
それらを含む開発の計画全体を**Meridian計画**としています.

Meridianの基幹部分は大きく以下の4つの要素から構成されます.  

① **Meridim** : Meridianの仕様に基づいた中間プロトコルのデータ配列.
② **Board** : Meridianに対応し, 各社のアクチュエーターやセンサーを制御できるハードウェア基板. 
③ **Console** : Meridian Boardと通信すると共に各種アプリケーションと連携するためのPC用ソフトウェア. 
④ **Editor**：ロボットの計算動作などを設定, 実行するための自由度の高いソフトウェア. 

これらの基本形を誰もが使える状態で公開し, 導入のためのわかりやすい教材が**整った状態で提供されることを初期の目標**とします. 

また, Meridian Flow Systemを中心として, さまざまなハードウェアやシミュレータなどの資産を使えるようにしていく**応用開発を第二の目標**とします. 
これらの開発は並行して行われます.

## 5. Technical Philosophy / 設計原則

#### Rationale / 存在目的
- Meridianの導入を通じて, 学習者が電子工作やコーディングなど, 誰もがつまづかずに**ロボット開発の基礎を楽しく効率よく学習できる**環境を提供します.  
- 学習者が自然科学や最先端の技術の学習にいち早く辿り着けることに貢献します. 

#### Coding Standards / 実装原則
- 基幹部分のコードは, **想定ユーザーが十分に理解できる状態で公開**します. これによりユーザーが機能を理解した上で, 自ら改造・拡張できるようにします. 
- 判読が難しいコーディングテクニックの使用は最小限とし, わかりやすさを最優先します 
- 使用するプログラミング言語は, マイコン部はArduinoフレームワークのC++, PC部はpython, C++, JAVA系とします. 

#### Technical Baseline / 基盤技術要件
- ESP32使用で**100Hz更新**のスピード, 1回のデータ量は**1400byte以内**を担保します. 
- **廉価なマイコンで使用できる**よう, コードはESP32で動作できるレベルの軽量さとします.
- 最小構成部分と, 機能拡張部分は分離可能な状態にします. 

#### Priority / 優先順位体系
標準策定および公式実装の判断は, 以下の優先順位に従います.
- **実機動作性＞全体最適性＞可読性＞軽量性＞拡張性＞合理性＞効率性**

#### Operational & Ethical Boundaries / 運用・倫理指針
- 上記は基本機能に限ったルールであり, 改良や拡張を想定しない応用プログラムについては, 自由に作成してよいとします. 
- 成果物が公序良俗やモラルに反しないこと, 人を傷つけないことを守ります. 
- 次期バージョンであるMeridian FLXについても, 基本は上記に従います. ただし, コーディングのレベルは解説を付けることや実機動作性を前提に高度であってもよいとします.

## 6. Goal / 目標

まず, Board, Console, Editor の基本となる開発環境を整備し, 市販のものなどマイコンベースの**既存のシステムと代替可能なレベルに達することを第一の目標**とします.  

また, 機能を拡張することで, **過去に販売されたロボットから最新のロボットまで, 同一のシステムで動かせる開発環境にすることを第二の目標**とします.  
これにはモーターのパラメータ同定など, 細かなチューニングも含みます.  

並行して, 機械学習により転ばずに走るヒューマノイドなど, 2025年までの**世界の研究成果を個人が手元で再現できるようになることを第三の目標**とします.  

最終ゴールは, 冒頭に記述した通り, Meridianを含んだ開発環境により, **科学を生涯にわたって楽しめる知的娯楽へと再定義**することです.

## 8. License & Monetization / ライセンスとマネタイズ

**基本部分は公共財, 拡張部分は市場原理を大原則**とし, プロジェクトの目的（学習スロープの社会実装）を妨げない範囲でマネタイズを行うことができます.

### License
- **基本部分はMITライセンスで公開**し, 無料で利用可能とします.
- 基本部分には他ソフト／他ハードとの**連携部分を含みます**.

### Monetization
ハードウェア・付加機能は, **基本機能を阻害しない範囲に限り開発者が権利を持ち**, マネタイズすることができます.  

例：Meridian準拠ボードの販売, Meridian対応ロボットの販売, UIやデザインを強化したアプリの販売, 基本機能に抵触しないプラグインの販売 等
